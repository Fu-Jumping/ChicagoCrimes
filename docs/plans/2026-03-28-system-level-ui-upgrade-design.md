# 系统级 UI 升级设计

## 背景

当前前端已经具备完整的桌面端分析骨架：`AppLayout.tsx` 负责顶栏、侧栏和状态栏，`AnalysisPageShell.tsx` 负责页面级标题区，`Dashboard.tsx`、`TrendAnalysis.tsx`、`TypeAnalysis.tsx`、`DistrictAnalysis.tsx`、`MapView.tsx` 负责 5 个主页面，`LineChart.tsx`、`BarChart.tsx`、`PieChart.tsx`、`RoseChart.tsx`、`CrimeHeatMap.tsx` 负责图表与地图渲染。

现阶段的主要问题不是功能缺失，而是视觉系统缺少统一层级：

- 左侧筛选项按控件堆叠，缺少明确的信息架构。
- 顶栏左上角品牌区名称冗长、结构生硬，无法体现“芝加哥犯罪可视化”的项目属性。
- 页面标题、副标题、说明文字之间层级不够清晰。
- 卡片大多仍是默认 `Antd Card` 模式，标题栏与图表内容区未分层。
- 顶栏和侧栏的信息密度偏低，界面显得空。
- 5 张背景图片尚未融入整体主题系统。
- 图表样式偏基础，缺少统一的高质量视觉语言。

本次升级的目标是构建一套“混合平衡型”界面体系：外层保留沉浸式情报中枢氛围，内层保持适合课程答辩和分析讲解的清晰秩序。

## 设计目标

1. 将左侧边栏升级为分区明确的分析控制台。
2. 统一全局品牌表达，替换现有不协调的左上角标题区。
3. 建立页面级标题、说明和模块标题的清晰文字层级。
4. 将分析卡片升级为“标题栏 + 内容舱”的模块化结构。
5. 把 `img/` 中的 5 张图片接入 5 个主页面背景，并确保深浅色模式下都不喧宾夺主。
6. 在不改动后端分析逻辑的前提下，提升图表和地图的视觉品质。
7. 保持现有的全局筛选状态、路由结构、请求逻辑和调试能力不变。

## 约束与实现边界

- 继续使用 Electron + React + Ant Design + D3 + React Leaflet，不引入新的大型 UI 框架。
- 视觉升级优先落在 `AppLayout.tsx`、`AnalysisPageShell.tsx`、全局 CSS、各视图和图表封装上。
- 筛选器行为保持现有 `GlobalFiltersContext` 语义不变，仅调整组织方式与文案。
- 顶栏不增加“系统状态标签”组件，避免信息过度堆叠。
- 背景图只作为最底层氛围层，卡片与下拉面板仍需保证数据可读性。

## 设计方向

整体风格采用“混合平衡型”：

- 外层：城市夜景与情报中枢氛围，通过背景图、轻微光晕、半透明边框建立记忆点。
- 内层：答辩级分析界面，通过严格的字号层级、卡片分区、网格和边界控制提高可读性。
- 结果：打开应用时能感受到项目气质，但开始讲解数据后，用户注意力仍然集中在筛选条件、标题与图表内容本身。

## 一、左侧边栏信息架构

左侧边栏从“连续堆叠的筛选器列表”升级为“分析控制台”，共分为 5 个区块：

### 1. 视图概览

位于侧栏最上方，显示：

- 当前页面小标签，例如“当前视图”
- 明显放大的页面主名称
- 当前页面一句话说明

该区块承担两个任务：告诉用户“我现在在哪个分析页面”，以及在侧栏里形成明显的视觉起点。

### 2. 时间范围

包含：

- 年份
- 月份
- 日期范围

该分组回答“正在观察哪段时间”，是所有筛选中最优先被识别的维度。

### 3. 事件条件

包含：

- 案件类型
- 是否逮捕
- 是否家暴

该分组回答“筛选的是哪类案件事件”。

### 4. 空间范围

包含：

- 分区
- 警区
- 选区
- 社区

其中“社区区域筛选”统一改为“社区筛选”，并在下拉选项、测试文案和说明文字中保持一致。

### 5. 筛选摘要

放在侧栏中下部，用轻量方式反馈：

- 当前启用的筛选数量
- 当前时间范围摘要
- 当前事件条件摘要
- 一键清空入口

这个区块让侧栏不仅是输入区域，也是当前状态的回显区域。

## 二、顶栏与左上角品牌区

### 品牌命名

顶栏左上角主品牌名称统一为：

- 中文主标题：`芝加哥犯罪可视化`

配套英文 eyebrow 使用更轻的辅助文本，例如：

- `CHICAGO CRIME VISUAL INTELLIGENCE`

不再使用“城市公共安全态势感知平台”这一类与当前项目主题不完全匹配、且过长的名称。

### 品牌区结构

左上角品牌区由 2 个部分组成：

1. 几何徽记
   使用 CSS 构建的简洁图形，形成识别符号，避免只剩一串文字。
2. 品牌文字组
   上方是小号英文标签，下方是中文主标题，形成清晰主次。

视觉上不采用生硬的直角矩形按钮，而是使用：

- 异形圆角信息牌
- 内外双层边框
- 轻微发光与半透明底板

### 顶栏整体布局

- 左侧：品牌区
- 中间：主导航
- 右侧：现有操作按钮（打印、主题切换）

右侧不增加新的状态胶囊组件，但通过品牌区与导航的重新排布提高顶栏密度，解决“顶栏偏空”的问题。

## 三、页面标题层级与模块卡片

### 页面标题三级层级

每个页面的头部统一拆成 3 级：

1. 页面标识（eyebrow）
   小号、低饱和度、标签化。
2. 主标题
   视觉重心，字号更大、字重更高。
3. 副标题/方法说明
   字号明显更小，行宽受控，用于讲清楚读图方式。

这套层级会同步应用到侧栏中的“当前视图”区域，解决“当前视图”和“视图名称”字号差距不足的问题。

### 模块卡片结构

所有图表卡片统一升级为两段式：

- 标题栏
  - 小标签或主题名
  - 主标题
  - 1 行短说明或筛选提示
  - 右侧操作区（例如对比年份选择器）
- 内容舱
  - 图表或地图
  - 数据态面板

这样每张卡片会从“普通容器”升级为“分析模块”，标题栏和图表内容区在背景、边框、透明度和内边距上形成明确区隔。

## 四、背景系统与主题策略

### 背景映射

`img/` 目录下的 5 张图片分别映射到 5 个主页面：

- 综合总览
- 趋势分析
- 类型分析
- 区域分析
- 空间地图

实现时保留 `img/` 作为原始来源，同时把图片复制到 renderer 的 Vite 资产路径中，以便稳定打包和按路由引用。

### 三层背景结构

每个页面背景由 3 层组成：

1. 原图层
   `cover` + `center`，保证桌面宽屏下完整铺满。
2. 主题蒙层
   深色模式叠加更深的蓝黑色渐变，浅色模式叠加柔和的白灰蓝渐变。
3. 统一纹理层
   轻微网格、噪点或径向光晕，用于把 5 张风格不同的图片重新拉回到同一套界面系统里。

### 透明度策略

- 顶栏与侧栏：更高不透明度，保证结构稳定。
- 页面头部与标题卡：中等透明度，兼顾氛围与信息权重。
- 图表内容舱：更透一些，但仍保留足够底色支撑坐标轴、图例与 tooltip。
- 浮层、下拉面板、日期面板：实体感更强，避免背景干扰交互。

结果是：背景图可见，但始终位于“场景层”，不参与争夺主视觉。

## 五、图表与地图视觉语言

### 折线图

- 主线条更平滑、更稳定。
- 折线下方增加向下渐隐的面积渐变。
- 悬停时显示高亮节点和辅助参考线。
- 多系列比较时，当前系列最突出，对比系列降低亮度但保留辨识度。

### 柱状图

- 柱体采用纵向渐变与轻微高光。
- hover 时单柱高亮，其余适度退后。
- 轴线和网格线统一收敛到更精致的对比度。

### 饼图与玫瑰图

- 统一色板，减少默认 D3 颜色感。
- 优化引导线与标签排布。
- 中心区域可加入总量或主题说明，提高完成度。
- 悬停时高亮当前扇区，弱化其他扇区。

### 地图

- 热力图配色更加层次化，避免默认热力层显得粗糙。
- 分区边界、tooltip、缩放控件和底图样式与整体主题统一。
- 地图卡片也遵循“标题栏 + 内容舱”的分析模块语言。

### 统一细节

- tooltip 升级为统一的卡片式浮层。
- 图例、对比选择器、标题栏操作区使用一致的字体和边框语言。
- 深浅色模式下共用同一套结构，仅切换色板、蒙层与对比强度。

## 验证标准

视觉升级完成后，至少满足以下标准：

1. 左侧边栏能够一眼区分“视图概览 / 时间范围 / 事件条件 / 空间范围 / 筛选摘要”。
2. 顶栏左上角品牌区不再出现生硬矩形按钮风格。
3. 页面主标题、副标题、说明文字的视觉层级明显优于当前版本。
4. 图表卡片均能看出标题栏与内容舱的分区。
5. 5 个主页面都有独立背景，且深浅色模式下都不影响读图。
6. “社区区域筛选”在界面中不再出现，统一为“社区筛选”。
7. 图表的 tooltip、渐变、图例和地图控件风格明显更统一。

## 非目标

- 不改动后端接口与分析查询逻辑。
- 不增加新的全局状态面板或系统状态标签。
- 不改变现有路由与页面数量。
- 不牺牲现有测试与调试面板能力来换取视觉效果。
