# 后端改进与 D3 协同规划说明（Spec）

## 1. 背景与目标

当前系统由 FastAPI 后端与 Electron + React + D3 前端组成，已具备基础分析能力。为提升分析深度、渲染性能与协同开发效率，本规划聚焦以下目标：

- 强化后端数据服务质量，提供稳定、可扩展、可追踪的分析接口
- 建立后端数据契约与 D3 渲染模型的一致性，降低前后端联调成本
- 提升图表交互体验与加载效率，支持更多维度与更大数据量

## 2. 规划范围

### 2.1 后端改进范围

- 统一分析接口返回结构，确保元信息、维度字段、数值字段语义稳定
- 增加查询参数规范（时间范围、筛选条件、分页与排序）
- 强化错误处理与异常码映射，减少前端兜底复杂度
- 引入性能优化策略（聚合索引建议、缓存层、慢查询定位）
- 增加数据质量校验（空值、异常值、时间字段一致性）

### 2.2 D3 协同范围

- 统一图表数据映射规范（分类轴、时间轴、数值轴、分组键）
- 明确图表组件输入协议（最小字段集、可选字段、空数据策略）
- 约定交互行为（筛选联动、悬浮提示、选区高亮、重置机制）
- 定义前端容错策略（接口超时、空结果、字段缺失降级展示）

### 2.3 非目标

- 本轮不涉及数据库选型变更
- 本轮不涉及 UI 视觉重设计
- 本轮不引入新的图表库替代 D3

## 3. 关键设计原则

- 契约优先：先稳定接口与字段定义，再推进组件迭代
- 兼容渐进：新增能力不破坏现有接口行为
- 可观测性：性能、错误、调用链可量化追踪
- 高内聚低耦合：后端聚焦数据服务，前端聚焦交互表达

## 4. 数据契约草案

建议统一响应结构：

```json
{
  "success": true,
  "message": "ok",
  "data": [],
  "meta": {
    "dimension": [],
    "metrics": [],
    "filters": {},
    "generated_at": ""
  }
}
```

字段约定：

- `data`：图表主数据数组，按图表类型约定键名
- `meta.dimension`：维度字段声明，如 `year`、`district`、`primary_type`
- `meta.metrics`：指标字段声明，如 `count`、`arrest_rate`
- `meta.filters`：本次查询生效的筛选条件
- `meta.generated_at`：服务端生成时间，用于调试与缓存判断

## 5. 协同流程

- 后端先发布接口契约草案并给出示例响应
- 前端基于契约开发 D3 组件适配层与空态/错态展示
- 双方按接口清单逐项联调，记录问题并回流到任务池
- 联调稳定后进行性能回归与体验验收

## 6. 里程碑

- M1：接口契约冻结与字段命名统一
- M2：核心分析接口优化完成并提供稳定测试数据
- M3：D3 图表组件完成契约适配与联动交互
- M4：联调收敛、性能达标、验收发布

## 7. 风险与应对

- 数据口径不一致：建立口径说明表并在接口元信息中显式声明
- 大数据量渲染卡顿：后端分桶聚合 + 前端采样与虚拟化策略
- 联调反复返工：采用契约版本号与变更日志机制
- 性能不稳定：引入慢查询监控与热点接口缓存
