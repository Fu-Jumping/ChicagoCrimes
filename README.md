# 芝加哥犯罪数据可视化（桌面客户端）

面向课程「数据库应用实践」的芝加哥开放犯罪数据集（Chicago Crimes）分析项目：数据存放在 **MySQL**，分析由 **FastAPI** 提供，**Electron + React + TypeScript** 桌面端负责交互与可视化。本文档侧重介绍**当前桌面客户端**的实现概况，以及相对课程基础要求之外、**从用户角度能直接感受到**的界面与流畅度上的提升。

---

## 技术栈概览

| 层级 | 技术 |
|------|------|
| 桌面壳 | Electron（electron-vite 构建）、无边框窗口 + 自定义标题栏 |
| 界面 | React 19、Ant Design 6、React Router（Hash） |
| 可视化 | **D3.js**（柱状图、折线图、饼图、玫瑰图等交互图表）、Leaflet + leaflet.heat（地理热力） |
| 通信 | Axios / `fetch` 调用本地 FastAPI（默认 `http://127.0.0.1:8000`） |
| 状态与体验 | 全局筛选 Context、主题 Context、`electron-store` 持久化向导状态 |
| 字体 | Space Grotesk + 中文字体回退（见 `App.tsx` 中 `ConfigProvider`） |

---

## 桌面客户端架构

- **主进程**（`frontend/src/main/index.ts`）：创建窗口、窗口控制 IPC、**系统文件对话框选择 CSV**（`select-csv-file`）、开发与生产环境下的后端启动策略（开发期由 `python start.py` 手动起后端；**打包安装后可在 resources 下拉起 `backend.exe`**，并将可写目录指向用户数据路径）。
- **预加载脚本**（`frontend/src/preload/`）：在 `contextIsolation: true` 下暴露安全的 `window.api`（窗口操作、选文件、设置向导 store 等）。
- **渲染进程**（`frontend/src/renderer/src/`）：单页应用，路由分为总览、趋势、类型、区域、地图等视图；所有分析页共享同一套**全局筛选**与**主题**。

## 课程要求之内（简要对应）

- **多维度分析**：年度/月度趋势、一周分布、24 小时分布、犯罪类型占比、区域/地点结构等，均以图表呈现。
- **多种图表形态**：柱状、折线、饼图/玫瑰图，且核心图表基于 **D3** 绘制，支持交互（符合「D3 + 后端」加分方向）。
- **地理展示**：地图页聚合热力与辖区态势，与全局筛选联动。

## 超出课程基础的体验与创新（用户视角）

在「能用数据库做统计、能画图」这一课设底线之上，本客户端更强调：**看起来像一款认真的桌面软件**、**长时间用着不费劲**、**大数据量下仍尽量顺手**。下面从**观感**和**流畅度**两方面说明（不涉及具体技术实现细节）。

### 界面与观感

- **整体风格**：深色 / 浅色一键切换；不同分析页面配有统一的版式与标题层次，并带有与页面主题呼应的背景氛围，避免「千篇一律的白底网页」观感。
- **窗口与布局**：自定义标题栏的桌面窗口形态；左侧可折叠的筛选栏集中管理时间、案件类型、区域等条件，各页共用同一套筛选，换页不用重新点选。
- **图表与地图**：柱状、折线、饼图等配色与主题联动，夜间模式下图表仍清晰可读；支持在同一图里对比多个年份，便于一眼看出变化；地图页以热力形式呈现空间聚集，与筛选条件联动。
- **时间浏览**：总览等页面提供直观的「时间轴式」月份切换，用滚轮或拖动即可在全年与各月之间浏览，比单纯下拉框更顺手。
- **中文体验**：界面与常见犯罪类型等以中文呈现为主，方便课堂展示与报告叙述。

### 流畅度与性能相关体验

- **后台预处理**：对常用统计在数据库侧做了汇总与索引（课设主线仍是原始明细入库），因此在前端切换年份、类型、区域时，**等待感明显减轻**，更接近日常使用的分析软件。
- **减少「点进去才加载」的空白**：进入软件后会**提前准备**各页常用的数据，减少第一次点开某页时的长时间白屏。
- **操作不「连点卡顿」**：筛选条件变化时，不会每一次微调都立刻打满请求，而是**稍作停顿再统一查询**，避免界面抖动和重复加载。
- **状态看得见**：加载中、没有数据、请求失败时都有明确提示，并可一键重试；若本机分析服务未启动，会给出**全屏说明与启动指引**，而不是无声失败。

### 首次使用与交付形态

- **首次设置向导**：在数据库尚未就绪时，按步骤完成连接配置、选择数据文件、导入与汇总即可使用，并可在遇到困难时查看手动说明或导出日志，降低「只有自己会配环境」的情况。
- **安装包交付**：可打包为 Windows 等平台的安装程序，本地安装后使用，而不仅限于在浏览器里打开一个网页。

如需了解底层技术栈与模块划分，见上文「技术栈概览」与「桌面客户端架构」。

## 快速开始

1. **准备数据**：将 `Crimes_-_2001_to_Present.csv` 置于 `data/`（大文件默认被 Git 忽略）。
2. **配置环境**：复制 `.env.example` 为根目录 `.env` 并按说明填写 MySQL（勿使用示例中的不安全默认组合）。
3. **一键开发启动**（仓库根目录）：

   ```bash
   python start.py dev
   ```

   将同时拉起后端与前端开发服务；仅前端时可进入 `frontend` 执行 `npm run dev`（需后端已监听 `8000` 端口）。

4. **首次导入**：若库表未就绪，桌面端会进入设置向导；亦可按用户指南手动执行 SQL 脚本。

**只开发后端**（不启动 Electron）：配置仍在**仓库根目录** `.env`，在 `backend` 目录执行 `python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload`；说明、环境变量表、测试与 SQL 脚本索引见 **[backend/README.md](backend/README.md)**。

更多部署与排错步骤见 **[用户指南](docs/user-guide.md)**。


## 构建桌面安装包

在 `frontend` 目录：

```bash
npm install
npm run build:win    # Windows；另有 build:mac / build:linux
```

构建前建议执行 `npm run typecheck` 或 `npm run gate:regression`（类型检查、测试与性能基线，见 `frontend/package.json`）。
